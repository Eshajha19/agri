# Security Policy

## 🌱 Project: Fasal Saathi

Fasal Saathi is a web application aimed at helping users with agriculture-related insights and services. We take security seriously and appreciate responsible disclosure of vulnerabilities.

---

## Supported Versions

We currently provide security updates for the latest deployed version:

| Version | Supported          |
| ------- | ------------------ |
| Latest  | ✅                 |
| Older versions | ❌         |

---

## Reporting a Vulnerability

If you discover a security issue, please report it responsibly.

### 📩 Contact
- Email: jha.esha2024@gmail.com

### ⏱ Response Time
- Acknowledgement within **48 hours**
- Regular updates during investigation

---

## 🔒 Responsible Disclosure Guidelines

Please follow these guidelines:
- Do **not disclose vulnerabilities publicly** until we fix them
- Do **not exploit the vulnerability beyond what is necessary to demonstrate it**
- Do **not access or modify other users’ data**

---

## 🛠 What to Expect

If your report is valid:
- We will investigate and fix the issue as soon as possible
- You may receive credit (if you want)

If not valid:
- We will explain why it does not qualify as a security issue

---

## 🚫 Out of Scope

The following are not considered security issues:
- Spam or social engineering attempts
- Denial-of-service attacks
- Issues requiring physical access to a device
- Reports without clear reproduction steps

---

## 🔐 Security Best Practices (For Contributors)

- Never commit API keys, tokens, or secrets
- Use environment variables (`.env` files)
- Regularly update dependencies
- Follow secure coding practices

---

Thank you for helping keep Fasal Saathi secure! 🌾
